Тест-кейс для позитивного сценария (успешная оплата при корректно введённых данных)

	
	TC-001

	Название: Успешная оплата банковской картой 

	Предусловия:

	 - Пользователь авторизован

	 - В корзине есть товар

	 - Открыта страница оплаты

	Шаги:

	 - Ввести корректные данные банковской карты (номер, срок действия, CVV)

	 - Нажать кнопку "Оплатить" 

	 - Пройти 3D Secure, если требуется

	Ожидаемый результат:

	 - Клиентская валидация проходит успешно

	 - Данные отправляются в платежный шлюз

	 - Платёжный шлюз возвращает статус "Успешно"

	 - Статус заказа меняется на "Оплачен"

	 - Пользователю отображается сообщение об успешной оплате

	 - На email отправляется чек



Тест-кейсы для негативных сценариев (отклонения оплаты банком и ошибки сервиса)

	
	TC-002

	Название: Оплата отклонена банком

	Предусловия:

	 - Пользователь авторизован

	 - В корзине есть товар

	 - Используется тестовая карта с отказом

	Шаги:

	 - Ввести данные карты, по которой банк должен отказать

	 - Нажать кнопку "Оплатить"

	Ожидаемый результат:

	 - Клиентская валидация проходит успешно

	 - Платёжный шлюз возвращает статус "Отклонено банком"

	 - Статус заказа не меняется на "Оплачен"

	 - Пользователю отображается сообщение об отказе

	 - Чек на email не отправляется


	TC-003

	Название: Ошибка платёжного сервиса

	Предусловия:

	 - Пользователь авторизован

	 - В корзине есть товар

	Шаги:

	 - Ввести корректные данные карты

	 - Нажать кнопку "Оплатить"

	 - Смоделировать ошибку платёжного шлюза (Например, отключение сервиса)

	Ожидаемый результат:

 	 - Пользователь получает сообщение об ошибке сервиса

	Статус заказа остаётся "Не оплачен"

	 - Чек не отправляется

	 - В логах фиксируется ошибка интеграции



Тест-кейсы для проверки граничных значений (проверка минимальной длины карты и проверки поля ввода сvv, если ввести две цифры)


	TC-004

	Название: Проверка минимальной длины номера карты

	Предусловия:

	 - Открыта страница оплаты

	Шаги:

	 - Ввести номер карты из 15 цифр, если требуется 16

	 - Заполнить остальные поля в соответсвии с валидацией

	 - Нажать "Оплатить"

	Ожидаемый результат:

	 - Появляется сообщение об ошибке (Неправильно введен номер карты)

	 - Оплата не отправляется в платежный шлюз


	TC-005

	Название: Проверка CVV из 2 цифр

	Предусловия:

	 - Открыта страница оплаты

	Шаги:

	 - Ввести корректный номер карты

	 - Ввести CVV из 2 цифр

	 - Нажать "Оплатить"

	Ожидаемый результат:

	 - Отображается ошибка валидации CVV (должно быть три цифры)

	 - Данные не отправляются в платежный шлюз


Тест-кейсы на проверку валидации (Пустые поля и неверный формат срока действия карты)



	TC-006

	Название: Пустые поля формы оплаты

	Предусловия:

	 - Открыта страница оплаты

	Шаги:

	 - Не заполнять поля карты

	 - Нажать "Оплатить"

	Ожидаемый результат:

	 - Отображаются сообщения о том, что поля обязательны

	 - Данные не отправляются в платежный шлюз


	TC-007

	Название: Неверный формат срока действия карты

	Предусловия:

	 - Открыта страница оплаты

	Шаги:

	 - Ввести корректный номер карты

	 - Ввести срок действия в неправильном формате (Например, 13/30 или 00/26)

	 - Нажать "Оплатить"

	Ожидаемый результат:

	 - Отображается ошибка формата даты

	 - Оплата не отправляется



Тест-кейсы на проверку интеграции с платежным шлюзом (передача данных и перенаправление на 3D Secure)

	TC-008

	Название: Проверка передачи данных в платежный шлюз

	Предусловия:

	 - Открыта страница оплаты

	 - Настроен доступ к логам или мок-сервису

	Шаги:

	 - Ввести корректные данные карты

	 - Нажать "Оплатить"

	 - Проверить запрос отправленный в платежный шлюз

	Ожидаемый результат:

	 - В запросе передаются корректные параметры (номер карты, сумма, валюта, id заказа)

	 - Ответ шлюза корректно обрабатывается системой


	TC-009

	Название: Проверка перенаправления на 3D Secure

	Предусловия:

	 - Используется карта требующая 3D Secure

	Шаги:

	 - Ввести корректные данные карты

	 - Нажать "Оплатить"

	Ожидаемый результат:

	 - Происходит перенаправление на страницу 3D Secure

	 - После успешного подтверждения пользователь возвращается в приложение

	 - Отображается корректный статус оплаты
